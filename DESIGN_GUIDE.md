# Pre.p Mood 레이아웃 시스템 디자인 가이드

## 🎯 핵심 원칙: 2트랙 독립 시스템

이 프로젝트는 **2개의 독립적인 트랙**으로 구성됩니다:

- **트랙 A: 레이아웃 트랙** (컨테이너/그리드/타이포 스케일)
  - 목적: 디자인 비율, 섹션 폭, 패딩, 그리드가 바뀌는 기준
  - 소유: `layout-system.css`
  - 기준: Mobile/Tablet/Desktop (767/768/1024 체계)

- **트랙 B: 헤더 트랙** (네비가 햄버거로 바뀌는 동작)
  - 목적: 헤더가 깨지기 전에 안전하게 모드 전환
  - 소유: `header.css` (단일 소유)
  - 기준: Header breakpoint X (기본 1024px, 필요 시 조정 가능)

### ⚠️ 중요: 두 트랙은 서로 독립적입니다

**컨테이너 브레이크포인트와 헤더 전환점은 별개입니다.**

예를 들어:
- **Tablet 구간 (768~1023px)**에서도 헤더는 모바일 모드(햄버거)를 유지할 수 있습니다
- 이는 **의도된 UX**이며, 명품 사이트들 대부분이 이렇게 동작합니다
- 태블릿은 레이아웃은 넓지만 헤더는 햄버거를 유지하는 경우가 많습니다

## 📐 트랙 A: 컨테이너 브레이크포인트 (레이아웃 트랙)

**소유 파일**: `layout-system.css`  
**목적**: 디자인 틀(스케일) 결정

### 1️⃣ Mobile (모바일)
- **화면 크기**: `0px ~ 767px`
- **컨테이너 최대 폭**: `420px`
- **좌우 패딩**: `20px` (작은 폰 360px 이하는 `16px`)
- **실제 콘텐츠 폭**: `380px` (420px - 20px×2)
- **타이포그래피**: `15px` 기본

**대표 디바이스:**
- iPhone SE (375px)
- iPhone 12/13/14 (390px)
- Galaxy S21 (360px)

**디자인 작업 시:**
- 피그마 아트보드: `420px` (컨테이너 기준)
- 또는 `375px` (iPhone SE 기준, 가장 작은 일반 모바일)
- 콘텐츠는 좌우 `20px` 여백 고려

---

### 2️⃣ Tablet (태블릿)
- **화면 크기**: `768px ~ 1023px`
- **컨테이너 최대 폭**: `840px`
- **좌우 패딩**: `32px`
- **실제 콘텐츠 폭**: `776px` (840px - 32px×2)
- **타이포그래피**: `15px` (모바일과 동일)

**대표 디바이스:**
- iPad (768px)
- iPad Pro 10.5" (834px)
- Galaxy Tab (800px)

**디자인 작업 시:**
- 피그마 아트보드: `840px` (컨테이너 기준)
- 또는 `768px` (iPad 기준)
- 콘텐츠는 좌우 `32px` 여백 고려

---

### 3️⃣ Desktop (데스크탑)
- **화면 크기**: `1024px~`
- **컨테이너 최대 폭**: `1280px` (Narrow) / `1680px` (Wide)
- **좌우 패딩**: `40px`
- **실제 콘텐츠 폭**: `1200px` (1280px - 40px×2)
- **타이포그래피**: `16px` (모바일/태블릿보다 큼)

**대표 디바이스:**
- 노트북 (1366px, 1440px)
- 데스크탑 모니터 (1920px, 2560px)
- 맥북 (1440px, 1680px)

**디자인 작업 시:**
- **본문**: `1280px` (컨테이너 기준)
- **헤더**: `1440px` 또는 `1680px` (컨테이너 기준, 스케일 전용)
- **히어로/비주얼**: `1920px` 또는 `2560px` (전체 폭 기준)
- 콘텐츠는 좌우 `40px` 여백 고려

## 🎛️ 트랙 B: 헤더 전환점 (헤더 트랙)

**소유 파일**: `header.css` (단일 소유)  
**목적**: 헤더 모드 전환 (햄버거 ↔ 풀 네비게이션)

### 헤더 전환점 X

**기본값**: `1024px`

**전환 기준:**
- **Header Mobile Mode**: `0px ~ (X-1)px` (햄버거 메뉴)
- **Header Desktop Mode**: `Xpx~` (풀 네비게이션)

**조정 방법:**
- 기본값은 `1024px`로 시작
- **실제 조정은 "헤더 깨짐 임계치 기반"으로 결정**
- 데스크탑 헤더가 처음 깨지는 폭을 실제로 측정
- 그 폭 + 여유 40~80px을 X로 확정

**예시:**
- 헤더가 1100px에서 깨지기 시작 → X = 1180px (1100 + 80)
- 헤더가 980px에서 깨지기 시작 → X = 1020px (980 + 40)

### ⚠️ 주의사항

1. **헤더 전환점은 컨테이너 브레이크포인트와 독립적입니다**
   - Tablet 구간(768~1023px)에서도 헤더는 모바일 모드를 유지할 수 있습니다
   - 이는 의도된 UX입니다

2. **헤더 관련 breakpoint는 `header.css`에서만 관리합니다**
   - 다른 파일(`global.css`, `responsive.css`, `page.css`)에서 헤더 breakpoint를 변경하면 안 됩니다
   - 헤더 모드 전환은 X 하나로만 통일합니다

3. **헤더 스케일 조정은 별도로 관리 가능합니다**
   - 헤더 높이/간격 조정은 "모드 전환"이 아니라 "스케일 조정"입니다
   - 예: `1500px`에서 헤더 높이를 줄이는 것은 스케일 조정으로 분류됩니다

### 화면 크기별 레이아웃 + 헤더 조합

| 화면 크기 | 레이아웃 트랙 | 헤더 트랙 | 설명 |
|----------|-------------|----------|------|
| **0~767px** | Mobile 레이아웃 | Mobile 헤더 (햄버거) | 모바일 레이아웃 + 모바일 헤더 |
| **768~1023px** | Tablet 레이아웃 | Mobile 헤더 (햄버거) | 태블릿 레이아웃 + 모바일 헤더 (의도된 UX) |
| **1024px~** (X 기본값) | Desktop 레이아웃 | Desktop 헤더 (풀 네비) | 데스크탑 레이아웃 + 데스크탑 헤더 |

**핵심**: Tablet 구간에서 헤더가 모바일 모드를 유지하는 것은 정상입니다.

## 🎨 디자인 작업 시 핵심 원칙

### ✅ DO (해야 할 것)

1. **3개 버전으로 디자인**
   - Mobile: 420px (또는 375px)
   - Tablet: 840px (또는 768px)
   - Desktop: 1280px (또는 1440px)

2. **Safe Area 고려**
   - 좌우 패딩을 제외한 실제 콘텐츠 영역에 디자인
   - Mobile: 좌우 20px 제외
   - Tablet: 좌우 32px 제외
   - Desktop: 좌우 40px 제외

3. **타이포그래피 고정값 사용**
   - Mobile/Tablet: 15px 기준
   - Desktop: 16px 기준
   - ❌ vw 기반 폰트 사용 금지 (명품 톤 유지)

4. **간격도 고정값**
   - breakpoint별로 고정된 간격 사용
   - ❌ clamp() 남용 금지

5. **헤더는 별도로 디자인**
   - 헤더 전환점 X(기본 1024px) 기준으로 2개 버전만 디자인
   - Mobile 헤더: 0px ~ (X-1)px
   - Desktop 헤더: Xpx~

---

### ❌ DON'T (하지 말 것)

1. **계속 변하는 디자인 금지**
   - 화면 크기마다 다른 디자인 ❌
   - 3개 breakpoint에서만 변경 ✅

2. **vw/clamp 남용 금지**
   - "화면 크기에 따라 계속 변하는" 느낌 ❌
   - "구간별로 정돈된" 느낌 ✅

3. **헤더와 레이아웃을 동일한 breakpoint로 묶지 말 것**
   - 헤더 전환점과 컨테이너 breakpoint는 독립적입니다
   - Tablet 구간에서 헤더가 모바일 모드를 유지하는 것은 정상입니다

## 🔍 Breakpoint 전환 시점 요약

## 🎯 "큰 모니터에서 왜 안 옹졸하냐?" (핵심 원칙)

명품 사이트가 큰 모니터에서도 옹졸하지 않은 이유는 **섹션별로 다른 컨테이너 폭**을 사용하기 때문입니다.

### ✅ 올바른 구조

```
┌─────────────────────────────────────────────────────────┐
│ 헤더: 1440px (또는 1680px) - Wide 컨테이너 (스케일 전용) │
├─────────────────────────────────────────────────────────┤
│ 히어로: 100% - Full-bleed                               │
├─────────────────────────────────────────────────────────┤
│ 본문: 1280px - Narrow 컨테이너                          │
├─────────────────────────────────────────────────────────┤
│ 상품 리스트: 100% 또는 1440px - Wide/Full              │
└─────────────────────────────────────────────────────────┘
```

### ❌ 잘못된 구조 (옹졸해 보임)

```
┌─────────────────────────────────────────────────────────┐
│ 모든 섹션: 1280px로 통일                                 │
│ → 큰 모니터에서 가운데로 옹졸하게 모임                  │
└─────────────────────────────────────────────────────────┘
```

### 적용 가이드

| 섹션 타입 | 컨테이너 폭 | 예시 |
|----------|------------|------|
| **헤더/네비게이션** | Wide (1440~1680px) | 로고, 메뉴, 아이콘 영역 (스케일 전용) |
| **본문 텍스트/폼** | Narrow (1280px) | 문의 폼, 약관, 정책 페이지 |
| **히어로/비주얼** | Full-bleed (100%) | 메인 배너, 룩북 이미지 |
| **상품 리스트** | Wide 또는 Full | 카탈로그, 상품 그리드 |

## 📌 개발자 참고사항

### 파일별 책임

- **`layout-system.css`**: 레이아웃 트랙 (컨테이너 breakpoint 767/768/1024)
- **`header.css`**: 헤더 트랙 (헤더 전환점 X, 기본 1024px)
- **`global.css`**: 전역 변수, reset, base 스타일
- **`page.css`**: 공통 컴포넌트
- **`responsive.css`**: 공통 컴포넌트 반응형 (헤더 제외)

### 헤더 전환점 X 조정 방법

1. 데스크탑 헤더가 깨지기 시작하는 폭을 실제로 측정
2. 그 폭 + 여유 40~80px을 X로 확정
3. `header.css`의 `@media` 쿼리에서 X 값 수정
4. 모든 페이지에서 동일하게 적용됨 (로드 순서 통일)

### 기존 혼재 breakpoint 처리

- **1199/1200/1500**: 헤더 관련이면 `header.css`로 흡수, 헤더 모드 전환은 X 하나로 통일
- **1500**: 헤더 높이/스케일 조정이면 "스케일 조정 구간"으로 분류 (가능하면 1280/1600으로 재정리)

## 🎯 최종 원칙 (한 줄 요약)

**컨테이너는 767/768/1024 체계를 유지하고, 헤더 전환점은 X(기본 1024px) 하나로 `header.css`에서 단일 소유하되, X는 '헤더 깨짐 임계치 기반'으로 최종 확정한다.**

이 구조를 통해:
- 디자이너와 합의가 쉬워집니다
- "수치 하나로 전 페이지 동시 전환"이 달성됩니다
- 헤더와 레이아웃이 독립적으로 관리됩니다
